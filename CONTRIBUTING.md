# Contributing to GIDEVO API Tool

## Ground rules

- Keep PRs focused
- Do not commit secrets or runtime data
- Update user docs when behavior changes
- Note verification steps in the PR

## Setup

See [INSTALL.md](INSTALL.md), [README.md](README.md), and [docs/how-to-use.md](docs/how-to-use.md).

## Before you open a PR

- Fresh install works
- Relevant tests/builds pass
- Docs still guide a first-time user
